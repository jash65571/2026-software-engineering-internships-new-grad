# Resume Keyword Guide for Software Engineering Roles

ATS (Applicant Tracking Systems) match resumes against the job description. Missing the right keywords means a strong candidate can still get filtered out before a human reads the resume.

## How to find the right keywords

1. Open the job description.
2. Highlight every noun phrase that names a tool, language, or concept.
3. Highlight every verb that names a responsibility (build, design, deploy, test, ship, debug).
4. Highlight any "must have" or "required" terms.
5. Map each one to a real line in your resume. If you have done it, name it. If you have not done it, don't fake it — find an adjacent skill instead.

## Frontend roles

React, Next.js, TypeScript, Redux, React Query, Tailwind CSS, accessibility (a11y), Storybook, Jest, React Testing Library, Webpack, Vite, performance optimization, Core Web Vitals.

## Backend roles

Node.js, Express, NestJS, Python, Django, Flask, FastAPI, Java, Spring Boot, Go, microservices, REST, GraphQL, gRPC, message queues (Kafka, RabbitMQ, SQS), caching (Redis), database schema design, indexing, query optimization.

## Full-stack roles

Combine frontend + backend lists, plus: end-to-end ownership, API design, authentication (OAuth, JWT, sessions), deployment (Vercel, AWS, Fly.io), monitoring (Sentry, Datadog).

## Data / infra roles

SQL, PostgreSQL, data modeling, ETL, Airflow, dbt, Snowflake, BigQuery, Spark, Kafka, Terraform, Docker, Kubernetes, Linux, observability.

## Generic engineering skills (almost every JD)

- Git, code review, pair programming
- CI/CD
- Unit and integration testing
- Agile, sprint planning, standups
- Cross-functional collaboration
- Technical writing, design docs
- Mentorship (if you have any)
