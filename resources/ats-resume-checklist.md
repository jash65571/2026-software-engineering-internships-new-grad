# ATS Resume Checklist for Software Engineering Jobs

Use this checklist before you apply.

## Basic checks

- One page if you are a student or new grad.
- PDF unless the employer asks for another format.
- Name, email, LinkedIn, GitHub, and portfolio are easy to find.
- Clear sections: Education, Experience, Projects, Skills.
- No tables, columns, or heavy graphics that may break parsing.
- No headers/footers with critical info (some ATS strip them).
- Standard fonts (Inter, Arial, Calibri, Georgia, Times).

## Keyword checks

For each job description, look for:

- Programming languages
- Frameworks
- Cloud and infra tools
- Databases
- Testing tools
- Role title keywords
- Required degree or graduation year
- Location or work type

## Common software engineering keywords

- JavaScript, TypeScript, Python, Java, Go, C++, C#, Rust
- React, Next.js, Vue, Angular, Node.js, Express
- Django, Flask, FastAPI, Spring Boot, .NET
- SQL, PostgreSQL, MySQL, MongoDB, Redis
- AWS, GCP, Azure
- Docker, Kubernetes, Terraform
- REST API, GraphQL, gRPC
- CI/CD, GitHub Actions, Jenkins
- Unit testing, integration testing, Jest, Pytest
- Git, code review

## Strong bullet format

`Built [thing] using [tech] to improve [result].`

Examples:

- Built a React dashboard in TypeScript and REST APIs so users could track application status in real time.
- Wrote a Python script that cleaned 10,000+ rows and reduced manual review time by 6 hours per week.
- Improved a SQL query by adding indexes and removing redundant joins, cutting p95 latency from 1.8s to 220ms.
